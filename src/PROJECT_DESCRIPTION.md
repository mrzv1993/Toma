# Описание проекта: Toma (Task Management "9-1")

## Обзор
Веб-сервис для управления задачами, основанный на методологии «9–1». Проект разработан на стеке React + TypeScript + Supabase и предназначен для повышения личной эффективности через строгую приоритизацию и учет времени.

## Методология «9–1»
Система организации задач в Спринте строится по принципу пирамиды приоритетов:
- **Уровень 9 (Высший приоритет)**: 1 задача
- **Уровень 8**: 2 задачи
- **Уровень 7**: 3 задачи
...
- **Уровень 1 (Низший приоритет)**: 9 задач

Всего в спринт можно взять ограниченное количество задач, распределенных по уровням важности.

## Структура интерфейса
Интерфейс разделен на 4 основные колонки, каждая из которых может быть свернута для экономии экранного пространства:

### 1. Hooks (Крючки)
Раздел для тематической группировки задач.
- **Функционал**:
  - Создание и редактирование "крючков" (групп/контекстов).
  - Отображение количества активных задач внутри крючка.
  - Подсчет суммарного затраченного времени на все задачи крючка.
  - Минималистичный дизайн: отображение количества задач вместо иконок.

### 2. Inbox (Входящие)
Накопитель для всех новых задач, которые еще не распределены в текущий спринт.
- Сюда попадают все созданные задачи по умолчанию.
- Отсюда задачи переносятся в Спринт.
- **Особенности**:
  - Задачи в Inbox нельзя отметить выполненными (отсутствуют чекбоксы).
  - Удаление задач возможно.

### 3. Sprint (Спринт)
Основное рабочее пространство для планирования текущего периода (например, дня).
- **Логика**:
  - **Строгая фиксация**: Задачи, добавленные в Спринт, **нельзя удалить или вернуть в Inbox**.
  - **Перемещение**: Возможно свободное перемещение задач между уровнями приоритета внутри спринта.
  - **Каскадное смещение**: При добавлении задачи на заполненный уровень, остальные задачи смещаются вниз (если позволяют лимиты уровней).
- **Выполнение и Таймер**:
  - Квадратные чекбоксы для отметки выполнения.
  - При отметке задачи выполненной активный таймер автоматически ставится на паузу.
  - **Блокировка таймера**: Нельзя запустить таймер на выполненной задаче (необходимо сначала вернуть статус "не выполнено").
- **История**: Возможность просмотра завершенных спринтов и их статистики.

### 4. Timer (Таймер)
Инструмент для трекинга времени.
- Запуск таймера для конкретной задачи.
- Учет затраченного времени (Spent Time).
- История активности.

## Технологический стек
- **Frontend**: React, TypeScript, Tailwind CSS
- **Backend**: Supabase (Auth, Database, Edge Functions)
- **State Management**: React Context API
- **UI Components**: shadcn/ui, Lucide React (иконки)
- **Drag & Drop**: Реализован для перемещения задач между колонками и уровнями.

## Ключевые особенности
- **Фокусировка**: Возможность сворачивать любую из 4-х колонок.
- **Учет времени**: Автоматический подсчет времени по задачам и крючкам.
- **Дисциплина**: Жесткая структура спринта не позволяет просто так удалять задачи из плана, мотивируя к их выполнению.
